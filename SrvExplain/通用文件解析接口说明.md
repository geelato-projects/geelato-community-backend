# 通用文件解析接口说明

面向调用者的接口调用说明：描述接口地址、入参/出参、返回结构示例与错误规范。

## 1. 通用返回包裹（ApiResult）

所有接口统一返回 `ApiResult`：
- `code`: number，成功为 `200`，失败通常为 `500`
- `status`: string，成功通常为 `success`，失败通常为 `error`
- `msg`: string，提示信息（失败时为错误原因）
- `data`: object|null，实际业务返回
- `cache`: boolean|null

成功示例：
```json
{
  "msg": "success",
  "code": 200,
  "status": "success",
  "data": {},
  "cache": null
}
```

失败示例：
```json
{
  "msg": "file type not supported: .TXT",
  "code": 500,
  "status": "error",
  "data": null,
  "cache": null
}
```

## 2. 接口列表

### 2.1 同步解析
- 地址：`POST /api/resolve`
- 说明：同步执行预设解析管道，返回最终结果 + 每步工件执行结果

### 2.2 异步提交
- 地址：`POST /api/resolve/submit`
- 说明：提交解析任务并返回 `taskId`

### 2.3 查询任务
- 地址：`GET /api/resolve/task/{taskId}`
- 说明：查询异步任务状态与结果（包含最终结果 + 每步结果）

## 3. 请求规范

### 3.1 Content-Type
- `POST /api/resolve`、`POST /api/resolve/submit`：`multipart/form-data`
- `GET /api/resolve/task/{taskId}`：普通 GET 请求

### 3.2 表单字段（POST 接口）

要求至少满足：`fileId` 与 `file` 其中一个存在。

| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| fileId | string | 否 | 平台附件 id |
| file | file | 否 | 文件字段（multipart） |
| biztag | string | 否 | 业务标识，用于决定走哪条预设管道；不传/不识别时按文件类型兜底 |
| config | string(JSON) | 否 | JSON 字符串，用于承载解析参数（可不传） |

### 3.3 config(JSON) 结构
config 为 JSON 字符串，作为业务侧可扩展配置，**与 biztag 强关联**：
- 服务端仅负责透传与解析为 JSON 对象，不对其内部字段做通用约束
- config 的具体结构、字段含义、校验规则、最终 result 的结构由对应 biztag 的业务约定决定

config 在 HTTP 层是一个 multipart 的 part（字段名为 `config`），推荐设置该 part 的 `Content-Type: application/json`。

## 4. 管道选择规则（biztag）

### 4.1 biztag 命名建议（业务语义）
biztag 应体现“实际业务场景”，例如：
- `carrier.so.parse`：船司 SO 文件解析
- `carrier.si.parse`：SI 文件解析
- `quote.sheet.parse`：报价单解析
- `booking.confirm.parse`：订舱确认书解析
- `invoice.parse`：发票解析

### 4.2 biztag 缺失/未知时的兜底
- 系统会根据文件类型走默认解析流程（具体实现对调用者透明）
- 若文件类型不受支持：返回失败

## 5. 调用示例

### 5.1 同步：上传文件（multipart），不传 biztag/config（走兜底）
```bash
curl -X POST "http://localhost:8080/api/resolve" \
  -F "file=@/path/to/demo.pdf"
```

### 5.2 同步：用 fileId 解析（走兜底）
```bash
curl -X POST "http://localhost:8080/api/resolve" \
  -F "fileId=YOUR_FILE_ID"
```

### 5.3 同步：船司 SO 文件解析
```bash
curl -X POST "http://localhost:8080/api/resolve" \
  -F "file=@/path/to/demo.pdf" \
  -F "biztag=carrier.so.parse" \
  -F 'config={"anyBizParams":{"example":"value"}};type=application/json'
```

### 5.4 同步：报价单解析
```bash
curl -X POST "http://localhost:8080/api/resolve" \
  -F "file=@/path/to/quote.xlsx" \
  -F "biztag=quote.sheet.parse" \
  -F 'config={"anyBizParams":{"example":"value"}};type=application/json'
```

### 5.5 异步：提交任务
```bash
curl -X POST "http://localhost:8080/api/resolve/submit" \
  -F "file=@/path/to/demo.pdf" \
  -F "biztag=carrier.si.parse" \
  -F 'config={"anyBizParams":{"example":"value"}};type=application/json'
```

返回示例（ApiResult.data 仅示意）：
```json
{
  "msg": "success",
  "code": 200,
  "status": "success",
  "data": { "taskId": "a1b2c3..." },
  "cache": null
}
```

### 5.6 异步：查询任务
```bash
curl -X GET "http://localhost:8080/api/resolve/task/a1b2c3..."
```

## 6. 出参结构（data）

### 6.1 同步解析 data：ResolveResponse
`POST /api/resolve` 成功时 `data` 返回：
- `result`: 最终产物（object|string|array|null）
- `steps`: 每步工件结果列表（array）

steps 单项字段（ResolveStepResult）：
- `artifactId`: 工件 id
- `success`: 是否成功
- `costMs`: 耗时（毫秒）
- `output`: 工件输出（可能为对象/字符串/长度等）
- `errorMsg`: 失败原因（success=false 时）

同步返回示例（仅示意，result/steps 取决于 biztag/文件类型兜底）：
```json
{
  "msg": "success",
  "code": 200,
  "status": "success",
  "data": {
    "result": { "someKey": "someValue" },
    "steps": [
      { "artifactId": "file.convert.pdf", "success": true, "costMs": 120, "output": "skipped", "errorMsg": null },
      { "artifactId": "file.publish", "success": true, "costMs": 15, "output": "http://xx/xx.pdf", "errorMsg": null }
    ]
  },
  "cache": null
}
```

### 6.2 异步提交 data：ResolveSubmitResponse
`POST /api/resolve/submit` 成功时 `data` 返回：
- `taskId`: string

### 6.3 查询任务 data：ResolveTask
`GET /api/resolve/task/{taskId}` 成功时 `data` 返回：
- `taskId`: string
- `status`: string（`pending`/`running`/`success`/`failed`）
- `biztag`: string|null
- `fileName`: string|null
- `createdAt`: number（毫秒时间戳）
- `updatedAt`: number（毫秒时间戳）
- `steps`: array（同 ResolveStepResult）|null
- `result`: object|string|array|null
- `errorMsg`: string|null

查询任务返回示例（仅示意）：
```json
{
  "msg": "success",
  "code": 200,
  "status": "success",
  "data": {
    "taskId": "a1b2c3...",
    "status": "success",
    "biztag": "carrier.si.parse",
    "fileName": "demo.pdf",
    "createdAt": 1710000000000,
    "updatedAt": 1710000001000,
    "steps": [
      { "artifactId": "file.publish", "success": true, "costMs": 10, "output": "http://xx/xx.pdf", "errorMsg": null },
      { "artifactId": "file.extract", "success": true, "costMs": 9000, "output": [{ "type": "text", "content": "..." }], "errorMsg": null }
    ],
    "result": [{ "type": "text", "content": "..." }],
    "errorMsg": null
  },
  "cache": null
}
```

## 7. 错误规范（msg 示例）
- 文件入参缺失：`fileId or file is required`
- 不支持的文件类型：`file type not supported: .XXX`
- biztag 未命中且文件类型无法兜底：`file type not supported: .XXX`
- 管道超时：`resolve pipeline timeout`
