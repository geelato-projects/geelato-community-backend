对应实现：

- 邮件 DTO：`cn.geelato.web.platform.srv.email.dto.*`

## 1. 基本约定

### 1.1 鉴权

均要求登录（后端通过 `SecurityContext.getCurrentUser()` 获取当前用户）。未登录时：

- JSON 接口：`ApiResult.fail("用户未登录")` / `ApiPagedResult.fail("用户未登录")`
- 文件下载：HTTP 401

### 1.2 通用返回结构

#### `ApiResult<T>`

```json
{"code":200,"status":"success","msg":"success","data":{}}
```

#### `ApiPagedResult<List<T>>`

```json
{"code":200,"status":"success","msg":"success","data":[],"total":0,"page":1,"size":20,"dataSize":0}
```

字段定义参考：

- [ApiResult](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-lang/src/main/java/cn/geelato/lang/api/ApiResult.java)
- [ApiPagedResult](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-lang/src/main/java/cn/geelato/lang/api/ApiPagedResult.java)

### 1.3 mailId 与 partId

- `mailId`：列表返回的 `id`，用于详情/附件/下载。编码：Base64Url(`${emailAccountId}:${folder}:${uidValidity}:${uid}`)，实现见 [MailIdCodec](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/MailIdCodec.java)。
- `partId`：附件序号字符串（"1"、"2"...），用于下载/保存附件。

## 2. 接口列表

### 2.0 获取文件夹列表（folder）

由于 `folder` 取值随邮箱服务端不同而不同，建议先调用本接口获取当前邮箱可用的文件夹名称，然后在列表/详情接口中传入。

- `GET /api/email/folder/list?emailAccountId={可选}&pattern={可选}`

Query 参数：

| 参数 | 必填 | 说明 |
|---|---:|---|
| emailAccountId | 否 | 邮箱账号ID；不传则取默认邮箱 |
| pattern | 否 | IMAP list pattern，默认 `*` |

返回：`ApiResult<List<EmailFolderDto>>`

`EmailFolderDto` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| name | string | 文件夹名称（非完整路径） |
| fullName | string | 文件夹完整名称（建议作为 folder 传参） |
| holdsMessages | boolean | 是否可存放邮件 |
| holdsFolders | boolean | 是否包含子文件夹 |

返回示例：

```json
{
  "code": 200,
  "status": "success",
  "msg": "success",
  "data": [
    {"name": "INBOX", "fullName": "INBOX", "holdsMessages": true, "holdsFolders": false},
    {"name": "Sent", "fullName": "Sent", "holdsMessages": true, "holdsFolders": false}
  ]
}
```

### 2.1 获取邮件列表（分页）

- `POST /api/email/message/pageQuery`

Body（JSON）：

```json
{
  "emailAccountId": "可选，不传取默认邮箱",
  "folder": "INBOX",
  "unread": true,
  "@p": "1,20"
}
```

folder 说明：

- `folder` 直接透传给 IMAP 的 `store.getFolder(folder)`，因此它的取值是“邮箱服务端实际存在的文件夹完整名称”（可能区分大小写、可能带层级分隔符）。
- 不同邮箱厂商/账号的文件夹名称可能不同，建议默认先用 `INBOX`，或使用邮箱账号配置中的 `imapFolderDefault`。
- 常见文件夹名称（仅示例，最终以目标邮箱为准）：
  - 收件箱：`INBOX`
  - 已发送：`Sent` / `Sent Messages` / `Sent Items`
  - 草稿箱：`Drafts`
  - 垃圾箱：`Trash` / `Deleted Messages` / `Deleted Items`
  - 垃圾邮件：`Junk` / `Junk E-mail` / `Spam`
  - 归档：`Archive` / `All Mail`
  - Gmail 常见：`[Gmail]/Sent Mail`、`[Gmail]/Drafts`、`[Gmail]/Trash`、`[Gmail]/Spam`、`[Gmail]/All Mail`

返回数据结构：`ApiPagedResult<List<EmailMessageListItemDto>>`

- DTO 字段见 [EmailMessageListItemDto](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/dto/EmailMessageListItemDto.java)
- 地址结构见 [EmailAddressDto](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/dto/EmailAddressDto.java)

返回示例：

```json
{
  "code": 200,
  "status": "success",
  "msg": "success",
  "data": [
    {
      "id": "ZTEyMzQ6SU5CT1g6MTIzNDU2Nzg5OjEwMA",
      "emailAccountId": "e1234",
      "folder": "INBOX",
      "subject": "报价单确认",

      "to": [{"name": "Bob", "address": "bob@example.com"}],
```

### 2.4 下载邮件附件（流式代理）

- `GET /api/email/message/{id}/attachment/{partId}/download`

返回：

- `Content-Type`：按附件 MIME 返回（或默认 `application/octet-stream`）
- `Content-Disposition`：`attachment; filename*=UTF-8''...`
- Body：附件二进制流

### 2.5 保存邮件附件到平台附件体系（生成 attachmentId）

- `POST /api/email/message/{id}/attachment/{partId}/save`

Body（可选）见 [SaveEmailAttachmentRequest](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/dto/SaveEmailAttachmentRequest.java)：

```json
{
  "serviceType": "ALIYUN",
  "sourceType": "email",
  "objectId": "可选，默认 mailId",
  "appId": "可选",
  "tenantCode": "可选"
}
```

返回数据结构：`ApiResult<SaveEmailAttachmentResult>`（见 [SaveEmailAttachmentResult](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/dto/SaveEmailAttachmentResult.java)）
