## 1. 基本约定

### 1.1 鉴权

均要求登录，后端通过 `SecurityContext.getCurrentUser()` 获取当前用户。未登录时：

- JSON 接口：`ApiResult.fail("用户未登录")` / `ApiPagedResult.fail("用户未登录")`
- 文件下载：HTTP 401

### 1.2 通用返回结构

#### `ApiResult<T>`

```json
{"code":200,"status":"success","msg":"success","data":{}}
```

#### `ApiPagedResult<List<T>>`

```json
{"code":200,"status":"success","msg":"success","data":[],"total":0,"page":1,"size":20,"dataSize":0}
```

字段定义参考：

- [ApiResult](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-lang/src/main/java/cn/geelato/lang/api/ApiResult.java)
- [ApiPagedResult](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-lang/src/main/java/cn/geelato/lang/api/ApiPagedResult.java)

### 1.3 `mailId` 与 `partId`

- `mailId`：列表返回的 `id`，用于详情、附件、下载、回复上下文。编码：Base64Url(`${emailAccountId}:${folder}:${uidValidity}:${uid}`)，实现见 [MailIdCodec](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/MailIdCodec.java)。
- `partId`：附件序号字符串，例如 `"1"`、`"2"`，用于下载或保存原邮件附件。

### 1.4 账号与厂商配置约定

- 一个用户可维护多个邮箱账号，通过 `defaultFlag=1` 指定默认账号。
- `providerCode` 目前支持：`qq`、`163`、`126`、`gmail`、`outlook`、`other`。
- 后端保存账号时会同时校验 `IMAP Host` 与 `SMTP Host`，因此前端即便使用厂商预设，也应提交最终的 IMAP/SMTP 实值。
- `authType` 常用值：
  - `auth_code`：授权码或密码登录
  - `oauth2`：OAuth2 配置
- 对于 `auth_code` 模式，`authSecret` 与 `smtpAuthSecret` 后端会加密存储。

## 2. 推荐接入顺序

建议 Web 邮件界面按下面顺序接入：

1. 邮箱账号管理：先新增并设默认邮箱账号。
2. 文件夹与收件箱：获取文件夹列表、邮件分页、详情、附件。
3. 联系人能力：接自动补全、常用联系人、最近联系人、回填。
4. 草稿与写信：接草稿保存、直接发送、草稿发送。
5. 回复/转发：通过 `composeContext` 预填主题、收件人、引用头和原文。

## 3. 邮箱账号接口

### 3.1 分页查询邮箱账号

- `POST /api/email/account/pageQuery`

Body 示例：

```json
{
  "emailAddress": "163.com",
  "defaultFlag": 1,
  "enableStatus": 1,
  "@p": "1,20"
}
```

说明：

- 支持 `emailAddress` 模糊过滤。
- `defaultFlag` 同时兼容 `defaultFlag`、`default_flag`、`isDefault`、`default`。
- 返回：`ApiPagedResult<List<UserEmailAccountDto>>`

### 3.2 获取邮箱账号列表

- `GET /api/email/account/list`

说明：

- 返回当前用户全部未删除邮箱账号，按默认账号优先排序。
- 返回：`ApiResult<List<UserEmailAccountDto>>`

### 3.3 获取邮箱账号详情

- `GET /api/email/account/{id}`

说明：

- 返回：`ApiResult<UserEmailAccountDto>`
- `UserEmailAccountDto` 中会回显 `providerCode`、`smtpHost`、`smtpPort`、`smtpSsl`、`smtpStarttls`、`smtpAuthUser`、`smtpFromName`、`signatureHtml`。
- 密文类字段不会明文回显。

### 3.4 新增或更新邮箱账号

- `POST /api/email/account/createOrUpdate`

Body 参考 [UserEmailAccountUpsertRequest](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/dto/UserEmailAccountUpsertRequest.java)：

```json
{
  "id": "可选，编辑时传",
  "emailAddress": "demo@163.com",
  "displayName": "demo",
  "providerCode": "163",
  "defaultFlag": 1,
  "imapHost": "imap.163.com",
  "imapPort": 993,
  "imapSsl": 1,
  "imapFolderDefault": "INBOX",
  "authType": "auth_code",
  "authUser": "demo@163.com",
  "authSecret": "授权码或密码",
  "smtpHost": "smtp.163.com",
  "smtpPort": 465,
  "smtpSsl": 1,
  "smtpStarttls": 0,
  "smtpAuthUser": "demo@163.com",
  "smtpAuthSecret": "可选，不填则前端可复用 authSecret",
  "smtpFromName": "demo",
  "signatureHtml": "<p>Best Regards</p>",
  "enableStatus": 1
}
```

说明：

- 新增成功返回邮箱账号 `id`。
- 当 `defaultFlag=1` 时，会自动取消当前用户其他账号的默认状态。
- 若 `authType=oauth2`，可以改为传 `oauth2Json`。
- 返回：`ApiResult<String>`

### 3.5 设置默认邮箱

- `POST /api/email/account/default/{id}`

说明：

- 把指定账号设为默认账号。
- 返回：`ApiResult<Boolean>`

### 3.6 删除邮箱账号

- `DELETE /api/email/account/{id}`

说明：

- 逻辑删除。
- 返回：`ApiResult<Boolean>`

## 4. 收件箱接口

### 4.0 获取文件夹列表

由于 `folder` 取值随邮箱服务端不同而不同，建议先调用本接口获取当前邮箱可用文件夹名称，然后在列表、详情、回填联系人等接口中传入。

- `GET /api/email/folder/list?emailAccountId={可选}&pattern={可选}`

Query 参数：

| 参数 | 必填 | 说明 |
|---|---:|---|
| emailAccountId | 否 | 邮箱账号 ID；不传则取默认邮箱 |
| pattern | 否 | IMAP list pattern，默认 `*` |

返回：`ApiResult<List<EmailFolderDto>>`

`EmailFolderDto` 字段：

| 字段 | 类型 | 说明 |
|---|---|---|
| name | string | 文件夹名称，非完整路径 |
| fullName | string | 文件夹完整名称，建议作为 `folder` 传参 |
| holdsMessages | boolean | 是否可存放邮件 |
| holdsFolders | boolean | 是否包含子文件夹 |

返回示例：

```json
{
  "code": 200,
  "status": "success",
  "msg": "success",
  "data": [
    {"name": "INBOX", "fullName": "INBOX", "holdsMessages": true, "holdsFolders": false},
    {"name": "Sent", "fullName": "Sent", "holdsMessages": true, "holdsFolders": false}
  ]
}
```

### 4.1 获取邮件列表

- `POST /api/email/message/pageQuery`

Body 示例：

```json
{
  "emailAccountId": "可选，不传取默认邮箱",
  "folder": "INBOX",
  "unread": true,
  "@p": "1,20"
}
```

`folder` 说明：

- `folder` 直接透传给 IMAP 的 `store.getFolder(folder)`，因此其取值是邮箱服务端实际存在的文件夹完整名称。
- 不同邮箱厂商、不同账号的文件夹名称可能不同，建议默认先使用 `INBOX`，或使用邮箱账号配置中的 `imapFolderDefault`。
- 常见文件夹名称仅供参考，最终以目标邮箱为准：
  - 收件箱：`INBOX`
  - 已发送：`Sent`、`Sent Messages`、`Sent Items`
  - 草稿箱：`Drafts`
  - 垃圾箱：`Trash`、`Deleted Messages`、`Deleted Items`
  - 垃圾邮件：`Junk`、`Junk E-mail`、`Spam`
  - 归档：`Archive`、`All Mail`
  - Gmail 常见：`[Gmail]/Sent Mail`、`[Gmail]/Drafts`、`[Gmail]/Trash`、`[Gmail]/Spam`、`[Gmail]/All Mail`

返回数据结构：`ApiPagedResult<List<EmailMessageListItemDto>>`

- `POST /api/email/contact/backfill`

Body 示例：

```json
{
  "emailAccountId": "e1234",
  "folders": ["INBOX", "Sent"],
  "messageLimit": 200,
  "includeTo": true,
  "includeCc": true,
  "includeFrom": true
}
```

说明：

- 会显式扫描最近邮件地址，沉淀到本地联系人表。
- 若不传 `folders`，通常建议至少扫描 `INBOX`，如需更好结果可加 `Sent`。
- 返回：`ApiResult<BackfillResult>`，结果中可用于展示新增/更新数量。

## 7. 发信与回复接口

### 7.1 直接发送邮件

- `POST /api/email/message/send`

Body 参考 [SendEmailRequest](file:///d:/geelato/geelato-enterprise/geelato-community/geelato-web-platform/src/main/java/cn/geelato/web/platform/srv/email/dto/SendEmailRequest.java)：

```json
{
  "emailAccountId": "e1234",
  "fromName": "demo",
  "to": [{"name": "Alice", "address": "alice@example.com"}],
  "cc": [],
  "bcc": [],
  "subject": "报价跟进",
  "textBody": null,
  "htmlBody": "<p>你好，附件请查收。</p>",
  "attachmentIds": ["1901234567890123456"],
  "mailAttachmentRefs": [
    {"mailId": "ZTEyMzQ6SU5CT1g6MTIzNDU2Nzg5OjEwMA", "partId": "1"}
  ],
  "composeMode": "reply",
  "sourceMailId": "ZTEyMzQ6SU5CT1g6MTIzNDU2Nzg5OjEwMA",
  "inReplyToMessageId": "<original-message-id@example.com>",
  "referencesHeader": "<ref1@example.com> <ref2@example.com>",
  "saveAsDraftOnFail": true,
  "draftId": "可选，若由现有草稿发送可传"
}
```

说明：

- 使用指定邮箱账号的 SMTP 动态发送。
- 发送成功后会更新联系人最近联系信息。
- 若 `saveAsDraftOnFail=true`，发送失败时可保留或回写草稿。
- 返回：`ApiResult<SendEmailResult>`

### 7.2 获取回复/转发上下文

- `GET /api/email/message/{id}/composeContext?mode=reply`

`mode` 常用值：

- `reply`
- `reply_all`
- `forward`

说明：

- 返回：`ApiResult<EmailComposeContextDto>`
- 接口会根据原邮件自动生成：
  - 建议主题，如 `Re:`、`Fwd:`
  - 推荐收件人、抄送人
  - `In-Reply-To`
  - `References`
  - 原文引用内容 `quotedText` / `quotedHtml`

## 8. 前端接入建议

### 8.1 账号配置页

- 先调用账号详情回显 `providerCode + IMAP/SMTP`。
- 厂商预设模式下，前端可隐藏手工输入区域，但提交时仍应带上 IMAP/SMTP 实值。
- `other` 模式需完整填写 IMAP/SMTP。

### 8.2 写信页

- 页面初始化可并行请求：
  - 默认邮箱账号列表
  - 常用联系人
  - 最近联系人
- 收件人输入联想时调用 `suggest`。
- 回复、回复全部、转发时调用 `composeContext`。
- 附件来源分两类：
  - 平台文件上传后得到 `attachmentId`
  - 原邮件附件通过 `mailId + partId` 引用

### 8.3 收件页

- 首次进入先调文件夹列表，再按选中文件夹调 `pageQuery`。
- 详情页点击附件可直接下载，或先保存到平台附件体系再复用到业务单据。
